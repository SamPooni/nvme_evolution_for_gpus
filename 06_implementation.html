06

Implementation Patterns

Practical guide to implementing GPU-direct storage access using GPUDirect Storage, kvikIO, and other optimization strategies.

Available APIs & Libraries

GDS
GPUDirect Storage
NVIDIA cuFile API
Native NVIDIA solution for direct NVMe-to-GPU transfers. Lowest latency, requires specific hardware support and driver configuration.
  • Direct PCIe P2P transfers
  • Zero CPU copies
  • Async and batch operations
  • CUDA 11.4+ required
kIO
kvikIO
RAPIDS / NVIDIA
High-level Python/C++ library that wraps GDS with automatic fallback to POSIX I/O when GDS unavailable. Best for portability.
  • Pythonic API
  • Automatic GDS fallback
  • NumPy/CuPy integration
  • Works without GDS hardware
ROC
ROCm Direct I/O
AMD
AMD's equivalent for Instinct GPUs. Similar P2P capabilities with HIP-compatible programming model.
  • HIP integration
  • MI200/MI300 support
  • Linux kernel 5.x+
  • Growing ecosystem

// Complete GDS workflow example #include <cufile.h> int main() { // 1. Initialize GDS driver cuFileDriverOpen(); // 2. Allocate GPU memory void* gpu_buf; cudaMalloc(&gpu_buf, BUF_SIZE); // 3. Register GPU buffer with GDS CUfileDescr_t cf_descr; cuFileBufRegister(gpu_buf, BUF_SIZE, 0); // 4. Open file with O_DIRECT (required!) int fd = open("/mnt/nvme/data.bin", O_RDONLY | O_DIRECT); // 5. Register file handle CUfileHandle_t cf_handle; cf_descr.type = CU_FILE_HANDLE_TYPE_OPAQUE_FD; cf_descr.handle.fd = fd; cuFileHandleRegister(&cf_handle, &cf_descr); // 6. Direct read: NVMe → GPU (the magic!) ssize_t bytes = cuFileRead(cf_handle, gpu_buf, BUF_SIZE, 0, 0); // 7. GPU kernel can now process data process_kernel<<>>(gpu_buf); // 8. Cleanup cuFileHandleDeregister(cf_handle); cuFileBufDeregister(gpu_buf); close(fd); cudaFree(gpu_buf); cuFileDriverClose(); return 0; }

Optimization Best Practices

Use Large I/O Sizes
Optimal performance at 1MB+ transfers. Coalesce small reads into large batches to amortize per-I/O overhead.
Align to 4KB
Both buffer addresses and file offsets must be 4KB aligned for O_DIRECT. Use cudaMallocHost() for aligned allocation.
Batch Operations
Use cuFileBatchIOSubmit() for multiple I/Os. Reduces per-operation overhead and improves SSD queue utilization.
Pre-Register Buffers
Registration has overhead. Register buffers once at startup, reuse them throughout the application lifetime.
Match NUMA Topology
Place NVMe and GPU on same NUMA node / PCIe root complex for best P2P performance. Use nvidia-smi topo -m to verify.
Use Async I/O
Overlap I/O with computation. Issue reads early, process previously loaded data while waiting for new data.
Common Pitfalls
  • Forgetting O_DIRECT flag — GDS requires direct I/O
  • Misaligned buffers — causes fallback to slow path
  • Not checking cuFileDriverGetProperties() — may silently fail
  • Using ext4 without proper mount options — use xfs or ext4 with dax

Call to Action

Addressing these challenges requires coordinated effort across the ecosystem. Here's what each stakeholder can do:

NVMe Consortium
Add GPU-native completion mode to NVMe spec. Define standard CMB layout for polling. Create GPU I/O profile in NVMe-oF.
GPU Vendors (NVIDIA, AMD, Intel)
Expand GDS/ROCm support. Publish P2P requirements clearly. Collaborate on cross-vendor storage APIs like kvikIO.
SSD Vendors
Implement CMB support. Optimize firmware for GPU I/O patterns. Validate GDS compatibility and publish results.
System Vendors
Design PCIe topologies for P2P. Enable ACS bypass for GDS. Provide clear documentation on GPU-storage connectivity.
Cloud Providers
Offer GDS-enabled instances. Work on secure multi-tenant GPU storage. Publish best practices for AI storage.
ML Framework Teams
Integrate kvikIO/GDS into data loaders. Optimize checkpoint I/O. Add storage-aware scheduling to training loops.

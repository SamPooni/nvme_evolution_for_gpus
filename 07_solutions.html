Solutions Architecture

Technology roadmap and solution architecture for GPU-optimized storage access. From today's GPUDirect Storage to tomorrow's CXL and Ultra Ethernet.

Technology Evolution Timeline

2020 - Present
GPUDirect Storage (GDS)
NVIDIA's PCIe peer-to-peer solution enabling direct NVMe→GPU DMA transfers. Eliminates CPU bounce buffer, reduces latency by 6-7×.
Production Ready CUDA 11+ PCIe Gen4/5
2023 - 2024
CXL 2.0 Memory Pooling
Compute Express Link enables shared memory pools with cache-coherent access. Type 3 devices provide memory expansion for GPU clusters.
Early Adoption 300-400ns latency DRAM-backed
2024 - 2025
PCIe Gen6 & NVMe 2.0
PCIe 6.0 doubles bandwidth to 256 GB/s (x16). NVMe 2.0 adds ZNS, KV, and improved multi-path support for AI workloads.
Sampling 2025 64 GT/s PAM4 Signaling
2025 - 2026
Ultra Ethernet Consortium (UEC)
AI-optimized Ethernet fabric replacing InfiniBand for training clusters. Sub-microsecond latency with lossless transport.
Specification 2025 800G-1.6T RDMA
2027+
CXL 3.0 + Persistent Memory
CXL 3.0 enables fabric-attached storage with memory semantics. True storage-class memory accessible via load/store.
Future Fabric Topology SCM/NAND
2028+
Unified GPU-Storage Fabric
Converged CXL + UEC fabric with native GPU support. Disaggregated compute and storage with microsecond access.
Vision UCIe Chiplets

CXL Memory Tier Architecture

CXL Reality Check

CXL 2.0 memory expansion is DRAM-backed with ~300ns latency — NOT flash storage. CXL-attached NAND storage (CXL.io) won't see significant latency improvements over NVMe because NAND access time (~70μs) dominates. CXL 3.0 fabric storage is 2027+ timeline.

Problem → Solution Mapping

Problem Solution
CPU bounce buffer adds latency and uses memory bandwidth GPUDirect Storage — PCIe P2P bypasses CPU entirely
GPU memory (HBM) capacity limited to 80-192 GB CXL Memory Pooling — Extends to multi-TB with ~300ns access
AI training needs checkpoints faster than NVMe can provide CXL DRAM tier — Checkpoint to CXL memory, async flush to NVMe
Random small I/O has high WAF and poor endurance ZNS/FDP — Application-controlled write placement
InfiniBand is proprietary and expensive at scale Ultra Ethernet (UEC) — Open standard with RDMA semantics
NVMe interrupt model hurts GPU SIMT execution Polling-based completion — GPU-initiated completion check
Disaggregated storage needs low-latency fabric CXL 3.0 Fabric — Memory semantics over switch topology (2027+)
